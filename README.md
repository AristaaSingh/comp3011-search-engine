# COMP3011 Search Engine Tool

A command-line search engine built for COMP3011 Web Services and Web Data, Coursework 2.

The tool crawls [quotes.toscrape.com](https://quotes.toscrape.com/), builds an inverted index of every word across all pages, and lets you search through them interactively. The index stores each word's frequency and positions on every page it appears on.

The tool works in two phases:
- **Build:** crawls the entire site (quote pages, author pages, tag pages), extracts visible text, and saves the inverted index to disk as a JSON file.
- **Search:** loads the saved index and lets you look up individual words or search for pages containing all words in a query.

## Index Data Structure

The index is an inverted index, meaning it maps words to pages rather than pages to words. This makes lookups fast: to find every page containing a word, you go directly to that word's entry rather than scanning through every page.

The index is saved as a JSON file (`data/index.json`) with the following structure:

```json
{
  "word": {
    "https://page-url/": {
      "frequency": 3,
      "positions": [4, 27, 61]
    }
  }
}
```

Each entry breaks down as:
- **word:** the lowercase token being indexed
- **page URL:** every page the word appears on, as a separate nested entry
- **frequency:** how many times the word appears on that page
- **positions:** the zero-based token indices where the word appears (e.g. position 4 means it is the 5th word on the page)

A real example for the word `"life"`:

```json
{
  "life": {
    "https://quotes.toscrape.com/": {
      "frequency": 3,
      "positions": [4, 27, 61]
    },
    "https://quotes.toscrape.com/page/2/": {
      "frequency": 1,
      "positions": [12]
    }
  }
}
```

The file is human-readable and can be opened and inspected directly alongside the code.

```

```

# Installation and Setup

**1. Clone the repository:**
```bash
git clone <your-repo-url>
cd comp3011-search-engine
```

**2. Create and activate a virtual environment:**
```bash
python -m venv myvenv
source myvenv/bin/activate
```

**3. Install dependencies:**
```bash
pip install -r requirements.txt
```

# Commands and Usage

Start the shell:
```bash
python -m src.main
```

You will see a `>` prompt. The four available commands are:

**build:** crawls the entire site, builds the inverted index, and saves it to `data/index.json`. This takes around 20 minutes due to the required 6-second politeness window between requests.
```
> build
Crawling site — this will take a while (~20 mins, 6s between requests)...
  [1] https://quotes.toscrape.com/
  [2] https://quotes.toscrape.com/page/2/
  ...
Done. 206 page(s) crawled, 8421 unique word(s) indexed.
```

**load:** loads a previously built index from `data/index.json`. Use this instead of `build` if the index has already been generated.
```
> load
Index loaded — 8421 unique word(s).
```

**print:** prints the full index entry for a word, showing every page it appears on with its frequency and positions.
```
> print life

'life' found on 5 page(s):

  https://quotes.toscrape.com/
    frequency : 4
    positions : [12, 57, 103, 210]
  ...
```

**find:** finds all pages containing a word, or all pages containing every word in a multi-word query (AND logic). Results are ranked by total frequency of the query words.
```
> find indifference

Results for 'indifference' — 1 word(s), 2 page(s) found:

  https://quotes.toscrape.com/
  https://quotes.toscrape.com/page/3/
```
```
> find good friends

Results for 'good friends' — 2 word(s), 1 page(s) found:

  https://quotes.toscrape.com/page/4/
```

# Testing

## Approach

Each component is tested independently using pytest, with each function tested in isolation rather than through the full pipeline. This makes it easier to pinpoint exactly where a failure occurs.

Key strategies used:
- **Mocking:** functions with external dependencies (network requests, `time.sleep`) are patched using `unittest.mock.patch` so tests run offline and instantly
- **Fixtures:** pytest's `tmp_path` fixture is used for file system tests, and shared `@pytest.fixture` functions build small controlled indexes for search tests
- **Controlled data:** indexer and search tests use hardcoded `PageData` objects with known text, so results are always predictable
- **Edge cases:** empty inputs, missing words, punctuation, case variations, duplicate query terms, and invalid file structures are all explicitly tested

## How to Run

**Full test suite:**
```bash
python -m pytest tests/ -v
```

**Single file:**
```bash
python -m pytest tests/test_crawler.py -v
python -m pytest tests/test_indexer.py -v
python -m pytest tests/test_search.py -v
```

**Offline only** (skips the 3 tests that make real network requests):
```bash
python -m pytest tests/test_indexer.py tests/test_search.py -v
```

## Test File Overview

**test_crawler.py** — tests all four functions in `crawler.py`

| Function | Approach | What is verified |
|---|---|---|
| `fetch_page` | Live network requests to quotes.toscrape.com | Valid URL returns content, 404 returns `None` without crashing |
| `parse_text` | Raw HTML strings passed directly | Tags stripped, output lowercased, script and style content excluded |
| `find_all_links` | Hand-crafted HTML with various link types | Relative to absolute conversion, external domains excluded, fragments stripped, no duplicates, `mailto:` and `javascript:` filtered out |
| `crawl_site` | `fetch_page` and `time.sleep` both mocked | Correct page count, URLs recorded, failed pages skipped, no URL revisited |

**test_indexer.py** — tests all five functions in `indexer.py`

| Function | Approach | What is verified |
|---|---|---|
| `tokenise` | Direct calls with various strings | Numbers excluded, contractions kept intact, hyphenated words split, empty input returns empty list |
| `add_page_to_index`, `build_index` | Small `PageData` objects with known text | Frequency counts, position lists, per-page tracking across multiple pages |
| Edge cases | Hardcoded inputs | Empty page text, punctuation-only text, mixed punctuation attached to words |
| `save_index`, `load_index` | `tmp_path` fixture for temporary files | Full round-trip, overwriting existing file, `FileNotFoundError` on missing file, `ValueError` on invalid structure |

**test_search.py** — tests all four functions in `search.py`

| Function | Approach | What is verified |
|---|---|---|
| `find_pages` | Shared fixture with a 3-page index | Missing words, empty query, mixed case, single and multi-word AND logic, exact matching, duplicate query words, multiple missing words all named |
| `print_word` | `capsys` fixture to capture printed output | Correct format (frequency, positions, URL shown), case insensitivity, empty input, surrounding whitespace stripped |
| `find_and_print` | `capsys` fixture | Each distinct output message produced correctly |
| `rank_results` | Separate fixture with known frequencies | Higher-scoring pages ranked first, lowest-scoring page always last |
